# Troble Shooting Log

## 2026-04-05

### next-env.d.ts 파일의 역할

#### [문제 현상]

- `Searchbar.tsx` 컴포넌트를 새 폴더(`components/Searchbar/`)로 이동시킨 후, 기존에 잘 작동하던 SVG 파일 임포트 경로에서 TypeScript 에러 발생.

#### [원인 분석]

- **환경 설정 파일 누락:** 프로젝트 루트에 있어야 할 `next-env.d.ts` 파일이 누락되어 있었음. 이 파일은 Next.js 환경에서 이미지 파일들을 TS가 이해할 수 있도록 타입을 선언(Declaration)해주는 역할을 함.

#### [해결 방법]

- **`next-env.d.ts` 파일 복구:** 프로젝트 최상위 경로에 파일을 생성하고 아래 내용을 추가함.
  ```typescript
  /// <reference types="next" />
  /// <reference types="next/image-types/global" />
  ```

#### [next.d.ts 파일이 무엇인가?]

- 공식문서: Next.js는 next-env.d.ts프로젝트 루트에 파일을 생성합니다. 이 파일은 Next.js 타입 정의를 참조하여 TypeScript가 코드 이외의 임포트(이미지, 스타일시트 등)와 Next.js 관련 타입을 인식할 수 있도록 합니다.

#### [파일명 해부]

next + .d + .ts

1. next (대상: 누구를 위한 것인가?)
   의미: 이 설정이 Next.js 프레임워크와 관련된 것임을 나타냅니다.

- 역할: Next.js는 일반적인 React와는 다른 독특한 기능들(이미지 최적화, SSR, 특수한 라우팅 등)이 많아요. 그래서 "이 프로젝트는 Next.js 환경이니까 그에 맞는 규칙을 적용해 줘!"라고 대상을 지정하는 이름표입니다.

2. .d (성격: 무엇을 하는 파일인가?)
   의미: Declaration(선언)의 약자입니다.

- 역할: 가장 중요한 포인트예요! 이 파일은 실행되는 코드(Logic)를 적는 곳이 아니라, "이런 타입들이 존재한다"라고 정의(Definition)만 내리는 곳입니다.

- 실제 브라우저에서 실행될 때는 이 파일의 내용은 싹 사라지고, 오직 개발 단계에서 TypeScript가 에러를 체크할 때만 참고합니다.

- cf) .config: 어떤 환경에서 어떻게 작동할 것인가를 담당하는 파일

3. .ts (형식: 어떤 언어로 적었는가?)
   의미: TypeScript 파일이라는 뜻입니다.

- 역할: 이 파일 안에 적힌 내용이 TypeScript 문법으로 작성되었음을 컴퓨터에게 알려줍니다. (만약 자바스크립트라면 .js였겠지만, 타입을 다루는 파일이니 당연히 .ts여야겠죠?)
